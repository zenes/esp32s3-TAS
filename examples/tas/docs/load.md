# 개발 준비 및 가이드라인 (Load Context)

이 문서는 개발 시작 전 로드하여 현재 프로젝트의 상태와 핵심 가이드라인을 확인하기 위한 용도입니다.

##  핵심 원칙
1.  **문서화**: 모든 `progress.md` 및 문서 업데이트는 반드시 **한글(Korean)**로 작성합니다.
2.  **이더넷 격리**: LCD 디버깅 중에는 `utilities.h`의 `ENABLE_ETHERNET` 정의를 해제하여 이더넷 코드가 실행되지 않도록 합니다.
3.  **하드웨어 구성**:
    *   **LCD**: SPI2 (HSPI) 채널 사용.
    *   **SD Card**: SPI3 (FSPI) 채널 사용 (활성화 시).
    *   **Ethernet**: W5500 (SPI 연결, TOE 기능 포함).

## 현재 상태
- **이더넷 격리 완료**: `tas.ino` 내 대부분의 이더넷 관련 코드 및 셸 명령어가 `#ifdef ENABLE_ETHERNET`으로 감싸져 있습니다.
- **빌드 환경**: Arduino Core 2.0.14 환경에서 빌드 가능하도록 호환성 패치가 적용되었습니다.
- **성능 목표**: WiFi-Ethernet 브리지 처리량 12~14 Mbps 이상 달성 (현재 9.8 Mbps 수준 유지).

## 작업 절차
1.  `/load` 스킬을 실행하여 최신 상태를 파악합니다.
2.  작업 완료 후 반드시 `/commit` 스킬을 사용하여 한글로 내용을 정리하고 푸시합니다.
