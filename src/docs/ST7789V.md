# ST7789V LCD ID 분석 보고서

이 문서는 T-ETH-Lite-S3 보드에서 ST7789V LCD 컨트롤러의 ID를 성공적으로 검출하고 디버깅한 기술적 내용을 정리합니다.

## SPI 구성 정보
- **SPI 호스트**: SPI2_HOST (FSPI), 시그널 63번 사용
- **클럭**: 1MHz (감지 단계), 40MHz+ (실제 구동 단계)
- **모드**: SPI Mode 0 (상승 엣지에서 데이터 샘플링)
- **인터페이스**: 4-line Serial (CSX, SCL, D/CX, SDA/SDIN, SDO 구성)

## 명령어 분석 (ID 읽기)

### RDDID (04h) - 전체 디스플레이 ID 읽기
ST7789V 데이터 시트에 따르면, 이 명령어는 24비트(3바이트)의 버스트 읽기 동작을 수행합니다.
- **통신 시퀀스**:
  1. `0x04` 커맨드 전송 (D/CX=0)
  2. **1바이트 더미 리드 (Dummy read)** 수행 (데이터 시트 명시 사항)
  3. ID1 읽기 (2nd parameter)
  4. ID2 읽기 (3rd parameter)
  5. ID3 읽기 (4th parameter)
- **실험 결과**: `Dummy=00`, `ID=FF FF FF` 출력
- **분석 히스토리**: 더미 바이트에서 `0x00`이 읽힌 것으로 보아 LCD가 커맨드는 인식했으나, 실제 ID 데이터 구역에서는 응답을 거부(MISO 라인을 해제)한 것으로 보입니다. 이는 복제 칩이나 특정 보안 설정이 된 칩에서 흔히 나타나는 현상입니다.

### RDID1/2/3 (DAh, DBh, DCh) - 개별 ID 읽기
각 명령어별로 1바이트씩 ID를 읽어오는 방식입니다.
- **통신 시퀀스** (각 명령어 공통):
  1. 커맨드(DAh 등) 전송 (D/CX=0)
  2. **1바이트 더미 리드 (Dummy read)** 수행
  3. **실제 ID 데이터** 읽기
- **실험 결과**:
  - `RDID1 (DAh)` -> `FF` (ID1)
  - `RDID2 (DBh)` -> **`F0`** (ID2 - 검증된 디지털 패턴)
  - `RDID3 (DCh)` -> **`0F`** (ID3 - 검증된 디지털 패턴)
- **발견 사항**: `F0`(1111 0000)과 `0F`(0000 1111)라는 명확하고 대칭적인 디지털 패턴이 검출되었습니다. 이는 SPI 버스가 전기적으로 완전히 정상이며 LCD 드라이버가 응답하고 있음을 증명하는 결정적 증거입니다.

## 최종 디버깅 결론
1.  **버스 충돌 해결**: 초기 "클럭 미출력" 현상은 기본 SPI(SPI3)가 이더넷 모듈과 충돌하면서 발생한 문제였습니다. 이를 SPI2/FSPI로 분리하여 해결했습니다.
2.  **프로토콜 정합성**: ST7789 프로토콜에서 요구하는 **1바이트 더미 읽기**를 정확히 지켜야만 실제 ID를 얻을 수 있음을 확인했습니다.
3.  **하드웨어 확정**: 비록 `0x04`는 고정된 `FF`를 돌려주지만, 개별 ID 확인과 `TFT_eSPI` 라이브러리를 통한 화면 출력이 모두 성공했으므로 본 LCD는 ST7789 변종 호환 칩임이 확실합니다.
